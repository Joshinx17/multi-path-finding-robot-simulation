A* ALGORITHM

A* is an informed search algorithm that finds optimal paths using a best-first strategy guided by a heuristic function.

f(n) = g(n) + h(n)

g(n) — exact cost from start to node n
h(n) — Octile distance heuristic to goal

A min-heap priority queue ensures each node is always expanded in order of lowest estimated total cost, guaranteeing optimality.

COOPERATIVE PATHFINDING

Each robot plans a path in the joint time-space domain — (col, row, timestep) — so it can reason about where other robots will be in the future.

A shared reservation table stores every cell every higher-priority robot will occupy at each timestep. Lower-priority robots route around these reservations — or wait in place.

Swap collisions (A→B simultaneously as B→A) are explicitly detected and blocked.

OBSTACLE PLACEMENT

During setup, click any non-corner cell to toggle an obstacle. Obstacles are represented as impassable walls. The simulation checks at start time whether each robot has at least one viable static path — if not, the robot is marked trapped.

DIAGONAL MOVEMENT

Robots can move in all 8 cardinal and diagonal directions. Diagonal cuts through wall corners are prevented — both adjacent orthogonal cells must be free — ensuring physically realistic traversal through narrow passages.

PRIORITY & COLLISION

Robots plan in priority order (R1 first, R4 last). A robot may wait in-place for one or more timesteps to yield to a higher-priority agent. Once all robots reach their opposite corners, the simulation completes.
