01
A* Pathfinding

A* finds optimal paths through informed search, balancing known travel cost with an admissible heuristic estimate to the goal.

f(n) = g(n) + h(n) · 0.8

g(n) is the exact accumulated cost from start to node n. h(n) is the Octile distance heuristic — accurate for 8-directional grids. The 0.8 weight makes h slightly inadmissible, prioritising speed over strict optimality in the cooperative planner.

A binary min-heap priority queue ensures each node expansion costs O(log n), keeping the search fast even on dense grids.

02
Cooperative Planning

Each robot plans in time-space — a 3D graph of (column, row, timestep). This lets lower-priority robots reason about where others will be and route around them.

A shared reservation table stores every cell occupied by higher-priority robots at each future timestep. Lower-priority robots treat reserved cells as temporary walls — or wait in place for one step to yield.

Swap collisions (A→B at the same instant B→A) are explicitly detected and blocked.

Optimal Order Search

Priority order matters — a bad ordering can trap lower-priority robots. This simulation tries all 24 permutations of planning order and selects the one that maximises robots arriving, then minimises total path cost.

Obstacle Placement

During setup, click any non-corner cell to toggle a wall. Walls are impassable. At runtime, a static A* check verifies each robot has at least one viable path — if not, the robot is flagged as trapped before the simulation begins.

Diagonal Movement

Robots traverse all 8 directions including diagonals. To prevent clipping through wall corners, a diagonal move is only permitted if both orthogonally adjacent cells are free — ensuring physically realistic navigation in tight spaces.

Conflict Resolution

Robots occupying the same cell at the same timestep is forbidden. Swap collisions are blocked. A robot may wait in-place (zero-displacement move) for any number of steps to yield. The best-order search maximises successful arrivals globally.
